# AI Engineering Collaboration Rules

## 工作原则

1. 开发前先分析项目结构和技术栈。
2. 优先遵循当前项目已有架构、命名和代码风格。
3. 优先最小可行实现，避免无意义重构。
4. 高风险修改先说明影响范围。
5. 不伪造运行结果、截图、PR 链接或测试结果。

## 代码分析流程

1. 先阅读根目录规则文件，例如 `AGENTS.md`、`CLAUDE.md`、`README.md`。
2. 使用关键词定位关键链路。
3. 从入口函数追到 service，再追到数据库 query。
4. 文档中必须写清楚文件路径和行号。

## 测试要求

1. 能跑测试就跑测试。
2. 未运行的检查必须说明原因。
3. 测试输出应保留在交付说明中。
